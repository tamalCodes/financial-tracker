# DATA_MODEL — next-ver

Post **mobile redesign**. Migrations `supabase/migrations/001..005`:
`001_mobile_redesign`, `002_profiles_opening_balance`, `003_rls`, `004_expense_tag`,
`005_bill_emi`. Schema in `supabase/schema.sql`. Plan:
[features/mobile-redesign.md](./features/mobile-redesign.md). All tables are owned per-user
via `user_id`.

## Ownership rule
Every row belongs to a user. Every query filters `.eq("user_id", userId)` — ownership is
enforced in the query layer. **RLS** (`003_rls`) is also enabled on the user tables as
defense-in-depth (DECISIONS D21), but code never relies on it alone.

## Month key
A "month" is the string `YYYY-MM-01` (day always `01`). Generated by `formatMonthKey`,
shifted by `shiftMonthKey(key, ±1)`. Stored in `credits.month`, `expenses.month`,
`investments.month`, `bills.month`.

## Money model — cumulative "Left in bank" (NEW)
The old per-month starting/closing balance is **gone**. `monthly_balances` is **deprecated and
unused** (table kept for back-compat). Balance is computed on read:

```
earned_m    = Σ credits(month)
spent_m     = Σ expenses(month) + Σ (bills paid in month)
invested_m  = Σ investments(month)          // per-month investment FLOW
leftInBank(month) = opening_balance + Σ_{m ≤ month} ( earned_m − spent_m − invested_m )
```
- `opening_balance` (`profiles`, migration 002) is the **one-time signup bank balance** — a
  starting point, **not income**; it seeds Left-in-bank but never a per-month tile (DECISIONS D16).
- The three HeroBalance tiles show the **current month only** (`earned_m`, `spent_m`, `invested_m`).
- "Left in bank" is **cumulative** across every month ≤ current — it piles up over time.
- **No carry-forward** (credits/expenses no longer copy rows; `carry_forward`/`carried_from_month`
  columns retained but unused).

### `profiles` — per-user profile (migration 002)
`user_id (PK), opening_balance numeric default 0, …`. `opening_balance` is captured once at
signup and feeds the cumulative Left-in-bank base (D16). Not a monthly mutation target.

## Tables

### `credits` — income entries (per month)
`id, user_id, month, description, amount, created_at`. `carry_forward`/`carried_from_month`
retained but **DEPRECATED/unused**. Feeds `earned_m` + Left-in-bank.

### `expenses` — spending entries (per month)
`id, user_id, month, description, amount, category, tag, created_at`. Plus deprecated
`carry_forward`/`carried_from_month`/`tags` (array — **never migrated to the live DB**; D18).
**`category`** ∈ `food | shopping | transport | health | groceries | other` (default `other`) —
drives the mobile pill + AddSheet picker. **`tag`** (migration 004) is a single nullable
free-form label set via the mobile edit modal (D18); empty clears to null. Feeds `spent_m` +
Left-in-bank. Recent-payments list is paginated (D20): dashboard returns page 1 only, 2+ via
`GET /api/expenses?page=`.

### `investments` — per-month investment FLOW
`id, user_id, month, description, amount, created_at`. Legacy
`start_month`/`is_active`/`carry_forward` retained but **DEPRECATED/unused** (recurring +
soft-delete model removed; hard delete now). Feeds `invested_m` + Left-in-bank.
> Distinct from the Investments **panel** (holdings/sips/portfolio_totals), which is manual
> display-only reference and does **not** affect the money model.

### `bills` — Bills & EMIs (separate ledger)
`id, user_id, month, name, amount, due_date (text), paid (bool), created_at` + EMI columns
(migration 005) `emi_id (uuid), emi_seq (int, 1-based), emi_months (int), emi_total (numeric)`.
A **paid** bill counts toward `spent_m` + Left-in-bank; unpaid does not. Paying a bill does
**not** create an expense row, and bills never appear in the Recent-payments list. **No overdue
state.**
- **One-off bill:** all `emi_*` null.
- **EMI (D17):** N rows sharing one `emi_id`, one per month from the start month, `emi_seq`
  1..N. Each is a normal bill — paying installment *k* marks that month's row paid and counts
  toward that month's spend. `emi_total` is the loan total (display only; `monthly × months`
  may exceed it due to interest). Index `bills_user_emi_idx (user_id, emi_id)` for rollup.

### `holdings` — portfolio reference (manual)
`id, user_id, kind ('fd'|'mutual_fund'), name, current_value, rate (FD), maturity_date (FD),
created_at`. Display-only; no money-model effect.

### `sips` — active SIPs (manual)
`id, user_id, name, monthly, due_date, paid_total, created_at`. Static reference; not cumulative;
no money-model effect.

### `portfolio_totals` — manual hero number
`user_id (PK), value`. One row per user; the portfolio value shown on the Investments panel.

### `monthly_balances` — DEPRECATED
`id, user_id, month, starting_balance, closing_balance`, unique `(user_id, month)`. **Unused**
since the redesign — kept only for back-compat. Do not read or write it.
